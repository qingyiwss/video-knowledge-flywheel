> 核心原则：**AI 漫剧不是一键生成的，是一段一段搭出来的。角色一致性不是写出来的，是管出来的。**

## 二、AI 友好型写作规则

### 2.1 剧本→视觉：铁律

**每条描述必须可直接转化为画面。** AI 不理解抽象叙事。

| ❌ 写这个 | ✅ 写这个 |
|----------|----------|
| 他感到深深的绝望 | 他低下头，眼眶泛红，右手攥紧衣角 |
| 办公室里气氛紧张 | 日光灯闪烁，两人隔桌对视，无人说话 |
| 她经历了内心的转折 | 她停下脚步，回头看了一眼，眼神由迷茫变为坚定 |

### 2.2 剧本格式标准

```
【场景1】办公室 · 夜晚 · 室内
中景 | 暖黄顶光 | 45°俯角
人物：林晨
动作：盯着屏幕，眉头紧锁
表情：焦虑，嘴唇紧抿
对白（林晨）：「修复不了……已经来不及了。」
时长：4秒
```

**强制字段**：场景 · 环境 · 机位 · 人物 · 动作 · 表情 · 台词 · 时长

### 2.3 每集节奏公式

```
0-3秒   视觉钩子（反常画面/悬念/冲突暗示）
3-15秒  展开冲突
15-25秒 反转/高潮
25-30秒 开放性结局（留钩子）
```

> **反转必须在第 10 秒左右出现，给观众留下悬念感。**

### 2.4 剧本长度限界

| 视频总长 | 分镜数 | 每镜时长 |
|---------|--------|---------|
| 15 秒   | 4-6 个 | 2-5 秒 |
| 30 秒   | 6-10 个 | 2-5 秒 |
| 60 秒   | 10-16 个 | 3-6 秒 |

## 三、提示词工程规范

### 3.1 角色生成提示词模板

```
[画质词] [角色描述] [脸部细节] [服装细节] [光照] [构图] [比例]

示例：
电影级真实风格。一位28岁亚洲男性，黑色短发，下颌线条分明，深棕色眼睛。身穿白色衬衫，领口敞开。柔和的侧光从左侧打入。半身中景，直视镜头，9：16竖屏构图。
```

### 3.2 场景提示词模板

```
[场景类型] [主要元素] [光照] [氛围] [景别]

示例：
现代办公室。落地窗前一张原木办公桌，电脑屏幕亮着蓝色代码。清晨阳光从百叶窗缝隙透入。安静压抑的氛围。中景，平视。
```

### 3.3 图生视频提示词模板

```
[动作描述] [约束条件] [禁止项]

示例：
同一个角色缓慢转头面向镜头，嘴角微微上扬。保持相同的面部特征、白色衬衫、柔和侧光、电影风格。平滑控制动作。禁止：服装变化、面部变形、大幅镜头运动。
```

### 3.4 提示词铁律

| 规则 | 说明 |
|------|------|
| 🔒 固定风格词 | 全剧统一「电影级真实」「柔光」「9：16」等基础词 |
| 🔒 固定角色描述 | 角色外貌描述全剧一字不变 |
| ❌ 禁加新风格词 | 不加「动漫」「水彩」「赛博朋克」等破坏一致性 |
| ❌ 禁用模糊词 | 「漂亮」「好看」「帅气」→ 用「下颌分明」「鼻梁高挺」 |
| ⚠️ 动作克制 | 「缓慢转头」>「转身冲刺」，「嘴唇微动」>「大声吼叫」 |
| ⚠️ 环境一致 | 同场景的所有镜头用相同的光线/背景描述 |

## 四、角色一致性管理

### 4.1 角色资产清单

每个角色必须建立以下资产后才能开始生产：

- [ ] **角色卡**：年龄/身高/体型/脸型/发型/发色/瞳色/常穿服装/标志物/常用表情
- [ ] **定妆照（母图）**：正脸清晰、光照均匀、表情中性
- [ ] **多角度图**：正面/侧面/四分之三侧/半身/全身
- [ ] **服装清单**：每套至少 1 张参考图
- [ ] **常见表情**：喜怒哀乐各 1 张

### 4.2 三支柱原理

```
1. 视觉锚定 > 文字描描述
   → 每次生成都用母图做参考图，不全靠文字

2. 动作克制
   → 大动作 = 脸崩。小动效 = 稳定。
   → 头发轻动、衣摆飘、镜头缓推、嘴唇微动

3. 光照/环境一致性
   → 全剧统一光照方向、色温、时间
   → 同场景所有镜头用完全相同的光照描述
```

### 4.3 角色崩了怎么办

| 症状 | 原因 | 修复 |
|------|------|------|
| 脸变了 | 参考图不够清晰/特征不明显 | 换更清晰的母图，加大脸部描写占比 |
| 衣服变了 | 描述模糊 | 「白色衬衫」→「白色尖领长袖衬衫，扣子为银色」 |
| 年龄变了 | 光照变化影响皮肤质感 | 全镜头统一光照描述词 |
| 风格变了 | 混入了新风格词 | 严格审查提示词，简化到核心描述 |
| 远镜头崩 | 脸太小无法锚定 | 靠发型/服装/标志物识别，不靠脸 |

## 五、质量标准

### 5.1 逐镜验收清单

- [ ] 角色面孔一致（对比母图）
- [ ] 服装一致（对比服装参考图）
- [ ] 场景光照一致（对比场景库）
- [ ] 动作流畅无突变
- [ ] 无明显 AI 痕迹（手指/背景穿帮）
- [ ] 与上一镜头情绪/动作连续

### 5.2 成片验收清单

- [ ] 所有人声清晰可辨
- [ ] 字幕与语音同步
- [ ] BGM 不盖过对白
- [ ] 前 3 秒有钩子
- [ ] 结尾有悬念/留白
- [ ] 9：16 竖屏比例
- [ ] 无明显跳帧/卡顿

## 六、自动化生产流程

```
我（AI）负责：                  你负责：
① 世界观 + 角色库建设          ① 可灵/即梦登录授权
② 分集剧本 + 分镜表            ② 审核成片质量
③ 逐镜头提示词（CSV）          ③ B站发布
④ TTS 配音文件
⑤ FFmpeg 合成 + 字幕
⑥ 输出最终 MP4

全自动流水线：
剧本 → 提示词 → API 生图/视频 → TTS → FFmpeg 合成 → 成片
（全程在服务器完成，你只审片 + 发布）
```

## 七、素材命名规范

```
[剧名缩写]_[集数]_[类型]_[编号]_[描述]

示例：
NP_E01_STILL_01_lin-chen-closeup.png          # 静帧：林晨特写
NP_E01_VID_03_office-wide-push.mp4            # 视频：办公室广角推拉
NP_E01_VO_lin-chen_line05.wav                 # 配音：林晨第5句台词
NP_E01_SUB.srt                                # 字幕文件
NP_E01_FINAL.mp4                              # 最终成片
```

---

> **最后一句话：做 AI 短剧拼的不是模型，是「稳」。从第一条片就按这个规范走，第十条片才能不崩。**
