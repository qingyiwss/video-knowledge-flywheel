# 经验教训

> 每次踩坑后的结构化反思。

## [2026-06-17] 工具调研阶段

### 教训 1：AI 工具排名 ≠ 真实口碑
- **现象：** 搜索「AI 漫剧工具推荐」得到大量排名文章，排名靠前的漫小芽、灵绘AI 看起来专业
- **真相：** 全部是 SEO 软文矩阵。漫小芽邀请码制、零真实用户案例；灵绘AI 的「测评」文章末尾挂推广链接
- **规则：只信有 B站/知乎真实用户教程的工具。** 可灵、即梦、海螺、剪映都有大量 UP 主免费教程 → 可信。没出现在任何 B站 UP 主视频里的工具 → 不可信。

### 教训 2：角色一致性靠管理，不靠提示词
- **起初认知：** 写好提示词就能保持角色一致
- **实际真相：** AI 模型每次生成都重建角色，不记上次的脸。一致性需要：固定母图做参考 + 固定参数 + 固定风格词 + 动作克制 + 逐镜验收
- **规则：角色是资产，不是提示词。** 角色卡、三视图、表情集、服装库必须建立后才能开工。

### 教训 3：AI 短视频 = 分镜图 + 轻动效，不是动画
- **起初认知：** AI 应该能直接生成完整剧情动画
- **实际真相：** 本质是「静态图 + 图生视频轻动效」。大动作崩脸，小动效稳定。克制决定产能，不是审美。
- **规则：每镜时长 2-5 秒，动作选「缓慢转头/嘴唇微动/镜头缓推」。**

### 教训 4：剧本 ≠ 提示词
- **起初认知：** 把剧本直接扔给 AI 就能生成
- **实际真相：** 传统剧本的「心理描写」「氛围渲染」AI 看不懂。必须翻译成具体的动作、表情、光照、景别。
- **规则：每个场景描述必须可直接转化为画面，控制在 40-60 字。**

### 教训 5：先做 1 条片，再谈自动化
- **起初认知：** 先搭全自动流水线
- **实际真相：** 连手动都没做出一条合格的片，就谈自动化 = 过早优化
- **规则：试水期用手动可灵免费额度，验证 B 站反馈后再升级到 API 自动化。**
