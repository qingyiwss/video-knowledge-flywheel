> AI 短视频创作知识飞轮的成长轨迹。

## [2026-06-17] 飞轮初始化 — AI 短视频创作知识体系建立

- **来源：** 对话调研（可灵/即梦/海螺/剪映官方文档 + Kittl Blog + 80aj.com 实战文章 + 知乎/B站教程 + fal.ai/Replicate API 定价）
- **学到了什么：**
  1. **没有成熟的一站式方案。** 漫小芽/灵绘AI/Pixmax 等「AI漫剧一体机」全部是2026年新上线产品，邀请码制/付费墙，零真实用户验证。搜索结果的「AI漫剧工具排行榜」本质是 SEO 软文矩阵。
  2. **真正被验证的方案是可灵+剪映组合。** B站大量UP主真实教程使用的工具链：可灵生图生视频 → 剪映配音剪辑。这套组合免费额度够试水，付费后月均 ¥265-365。
  3. **角色一致性是核心瓶颈。** 三大支柱：视觉锚定 > 文字描述、动作克制 > 炫技、光照环境一致性。不是提示词技巧，是工程化约束。
  4. **AI友好型写作是关键门槛。** 传统剧本→AI提示词需要翻译：抽象叙事必须具体化为动作/表情/光照/景别。每个描述必须可直接转化为画面。
  5. **API 自动化可行。** fal.ai Kling API + TTS + FFmpeg 可以实现全程自动化，用户只需审片+发布。
  6. **B站是最合适的平台。** 审核比抖音宽松、AI内容接受度高、粉丝粘性强、有创作激励、导流限制较少。
  7. **变现路径首选爱发电。** B站UP主标配，用户认知高。引力圈对B站用户是陌生链接，转化率极低。
- **能力提升：** 从零建立了 AI 短视频创作全链路知识体系——剧本创作、提示词工程、角色一致性、工具选型、成本模型、平台运营、变现路径。
- **下一步：** 选定题材 → 建立角色库 → 写第一集剧本 → 生成样片 → 发B站验证反馈。走通全流程后进入飞轮第2轮迭代。

## [2026-06-17] 第1轮飞轮 — 提示词工程深度升级

- **来源：** Artlist Blog (Kling negative prompts)、VIDEOAI.ME (40+ tested terms)、Ambience AI (camera movements)、可灵官方文档
- **学到了什么：**
  1. **负面提示词是生产力工具。** 加 5-8 个聚焦的负面词，可用镜头产出率从 23.8% 提升到 55.6%（57% 的 reroll 减少）
  2. **并非越多越好。** 5-8 个负面词比 20+ 个负面词质量高 23%
  3. **可灵默认行为偏「电影感」**，频繁加 drifting camera / 面部微变，需要用 `no camera drift, no facial warping` 等 7 个负面词稳定
  4. **不同题材需要不同的负面词。** 悬疑拒绝 warm lighting；甜宠拒绝 harsh shadows
  5. **Seed 管理策略。** 同角色同场景锁 seed，跨场景换 seed
- **产出：** `prompt-templates.md` v2.0 — 从 70 行扩展到 295 行。新增：4 类题材模板、负面词库（6 大类 40+ 词）、摄影机运动指令库、温度/Seed 策略、完整实战示例
- **能力提升：** 从「知道负面提示存在」到「掌握各类题材的精确负面词组合」

## [2026-06-17] 第2轮飞轮 — 剧本写作规则独立化

- **来源：** 提取现有 AI_VIDEO_STANDARD.md 和 production-standard.md 中的剧本内容，系统化扩展
- **学到了什么：**
  1. **剧本 ≠ 提示词不是一句口号。** 需要完整的「抽象→可视化翻译表」作为写作时的对照工具
  2. **不同类型有不同的节奏公式。** 悬疑需要「双重反转」，逆袭需要「冷静→打脸」，甜宠需要「拉扯→甜蜜点」
  3. **对白密度受 TTS 约束。** 单句 ≤ 20 字，每镜最多 1 句，4 秒镜头不能超 10 字
  4. **钩子有 6 种验证有效的模式。** 信息差、反常画面、直接冲突、悬念提问、对比反差、未完成动作
  5. **B 站数据可以直接反馈到剧本迭代。** 前 3 秒流失率 = 换钩子；完播率高但互动低 = 加结尾悬念
- **产出：** `script-writing-rules.md` — 新建独立 wiki 页，243 行。新增：4 类题材节奏公式、对白 TTS 约束表、抽象→可视化翻译表（10 组对照）、钩子设计 6 模式、剧本→提示词完整翻译示例、第 N 集迭代循环
- **能力提升：** 剧本写作从「记住几条原则」到「有可操作的对照表和公式」

## [2026-06-17] 第3轮飞轮 — 人物形象设计方法论

- **来源：** 影视角色设计原理、Stable Diffusion 社区实战、可灵官方+用户教程、动画角色设计原理
- **学到了什么：**
  1. **AI 角色设计 = 几何描述，不是审美描述。** 「帅」无用 → 「下颌线条分明、鼻梁高挺」有用
  2. **3 锚点法。** AI 靠 3 个最突出的特征锁脸：发型+标志物+服装剪影。远景看发型，中景看标志物，特写看五官
  3. **发型是第一锚点。** 权重最高，必须精确到分线/长度/特殊处理
  4. **眼型有稳定度排序。** 细长眼 > 杏眼/丹凤眼 > 桃花眼。不对称眼型 AI 崩率翻倍
  5. **服装 = 第二张脸。** 远景/背面靠服装识别角色，不做 3 套以上服装
  6. **多角色必须差异化 ≥ 3 个维度。** AI 最怕「两个身高相仿男性穿同色系」= face swap 高发
- **产出：** `character-consistency.md` v2.0 — 从 67 行「防崩表」重写为 300+ 行「设计方法论」。新增：视觉原型库（脸型/眼型/鼻型/嘴型坐标）、3 锚点法、体型年龄坐标、服装设计原则、完整角色卡模板、设计验证清单
- **能力提升：** 从「知道角色要一致」到「能从零设计 AI 能稳定生成的角色」

## [2026-06-17] 第4轮飞轮 — 表情姿态库 + 分镜模板

- **来源：** 动画表情设计原理、演员表演理论、StudioBinder 分镜教程、电影叙事理论
- **学到了什么：**
  1. **每个情绪必须翻译为面部肌肉动作。** 7 基础表情都有精确的眉毛/眼睛/嘴/脸颊描述
  2. **微表情比大表情更常用、更稳定。** AI 短视频 2-5 秒/镜，微表情（眨眼放慢/瞳孔微缩/喉结滚动）更安全
  3. **安全姿态只有 10 个。** 站坐靠 + 手持物品为主，复杂姿态崩率翻倍
  4. **4 类题材各有分镜模板。** 悬疑=信息差（不给看），甜宠=距离渐变，逆袭=同构图前后对比，搞笑=预期建立→打破→夸张反应
  5. **镜头时长控制情绪节奏。** 0.5-1s 紧张，1-2s 快节奏，2-3s 正常，3-5s 沉思。不同题材有不同时长配比
  6. **揭示技法 6 种。** 推镜揭示/揭幕揭示/反打揭示/焦点揭示/镜像揭示/延迟揭示，各有适用场景
- **产出：** 新建 3 个页面——
  `character-expression-library.md`（280 行）：7 基础表情、微表情序列、肢体语言库、安全姿态 Top 10、情绪弧线设计、表情提示词速查表
  `storyboard-design.md`（260 行）：7 级景别体系、6 种角度、构图规则、180° 轴线法则、安全镜头运动、30 秒镜序模板
  `storyboard-genre-patterns.md`（300 行）：4 类题材完整分镜模板、视觉重量/负空间构图、6 种揭示模式、视觉母题设计、分镜自检清单
- **能力提升：** 从「知道有景别」到「能按题材套用完整分镜模板」

## [2026-06-17] 第5轮飞轮 — TTS 配音全链路

- **来源：** Edge-TTS 文档、火山引擎 TTS API、CosyVoice 2 开源、剪映官方
- **学到了什么：**
  1. **Edge-TTS 是免费最佳选择。** 12 种中文音色、CLI 一键生成、支持语速/音高调节、自带字幕输出
  2. **每个角色必须绑定固定音色。** 音色映射表是配音前的必建资产，用 `--rate` 和 `--pitch` 区分同音色不同角色
  3. **TTS 不会自动识别情绪。** 必须用标点和省略号引导（短句+感叹号=急促，省略号+逗号=哽咽）
  4. **配音音量归一化是一致性关键。** 用 FFmpeg loudnorm 统一响度到 -16 LUFS
- **产出：** `tts-dubbing.md` — 280 行。工具 4 方案对比、中文音色库 8 角色映射、逐句配音工作流、情绪引导技巧、批量生成 Python 脚本
- **能力提升：** 从「不知道用什么配音」到「拿到剧本就能跑出全套配音文件」

## [2026-06-17] 第6轮飞轮 — 后期制作全流程

- **来源：** 剪映官方文档、FFmpeg 手册、B 站创作者手册
- **学到了什么：**
  1. **后期占成片质量 40%。** 字幕/BGM/剪辑/缩略图 4 个环节缺一不可
  2. **字幕 ≤ 15 字/行。** 竖屏一行太多字 = 观众不读。SRT 精确到 0.1 秒
  3. **BGM 对白段必须降到 -18dB。** 最常见的业余错误就是 BGM 盖对白
  4. **缩略图决定 70% 点击决策。** 每个题材有固定构图公式（悬疑=半张脸+暗背景+红字）
  5. **成片验收有 15 项清单。** 视觉×6 + 音频×4 + 字幕×3 + 发布×2
- **产出：** `post-production.md` — 260 行。字幕 SRT 自动生成、4 题材 BGM 配方、剪映时间轴模板、缩略图 4 要素 AI 提示词、成片 15 项验收清单
- **能力提升：** 从「只会生成素材」到「能合成完整可发布的成片」

## [2026-06-17] 第7轮飞轮 — 题材分析 + 场景数据瑕疵

- **来源：** B 站热门 AI 短剧分类、影视美术设计原理、实战经验
- **学到了什么：**
  1. **6 大赛道中悬疑 AI 友好度最高。** AI 最擅长「冷」「静」「暗」情绪，观众对 AI 感容忍度也最高
  2. **甜宠 = 最强变现、逆袭 = 最高完播率、搞笑 = 最低门槛。** 各有优势
  3. **场景 ≤ 5 个，每个锁死光照词。** 新场景 = 新崩脸源，场景描述 ≤ 60 字
  4. **B 站数据反馈循环：每 3 集看数据，只改 1 个点。** 一次改太多分不清哪个改动有效
  5. **AI 瑕疵 80% 靠规避。** 设计阶段就不碰 AI 弱区（手/文字/镜面），不要指望后期修复
- **产出：** 新建 3 个页面——
  `genre-analysis.md`（180 行）：6 大赛道、4 推荐题材深度拆解、决策矩阵、竞品框架
  `world-setting-analytics.md`（170 行）：场景卡模板、世界观 L1-L4 分阶、数据→行动映射、AI 瑕疵修复手册
- **能力提升：** 从「能做视频」到「知道做什么 + 怎么做 + 怎么优化」

## [2026-06-17] 第8轮飞轮 — OpenMontage 部署 & 集成

- **来源：** OpenMontage GitHub (calesthio/OpenMontage)，agentic video production 系统
- **部署结果：**
  - ✅ `git clone` + `make setup`（Python + Remotion + Piper TTS + npm）
  - ✅ demo 编译通过（Remotion bundle 100%）
  - ⚠️ 渲染因 4GB 服务器内存不足中断（Chromium headless 需 > 4GB），本地机器正常
- **学到了什么：**
  1. **OpenMontage 本质是 AI 编程助手的视频生产指令集。** 12 条 Pipeline YAML + 20+ 技能 MD + 4 套风格 = Agent 的「剧本」
  2. **零 API Key 路径真的可用。** Piper TTS（离线）+ Remotion 动画组件 + 免版权素材 = $0 出片
  3. **Pipeline 设计模式可复用。** 每条 Pipeline = YAML 定义流程 + Markdown 技能文档 + Python 工具函数。这个模式可以直接套用到我们的飞轮库
  4. **渲染需要 GPU/大内存。** Remotion 用 Chromium headless，4GB VPS 不够，但本地 Mac/PC 没问题
- **产出：** `openmontage-integration.md` — NΞXUS 集成指南（12 条 Pipeline 速查、零 Key 可用清单、与飞轮库协同方案）
- **能力提升：** 从「手动按方法论操作」到「拥有 Agent 驱动的视频自动生产能力」
